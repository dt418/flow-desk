# Outgoing Webhooks (P1-4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or superpowers:executing-plans. Checkbox (`- [ ]`) tracking.

**Goal:** Workspace-scoped outgoing webhooks — register URL + secret + event filter, receive HMAC-signed POSTs on matching `TaskActivity` events, delivery log for retry + audit. Reuses shipped `TaskActivity` stream + F7 BullMQ infra.

**Design spec:** `docs/superpowers/specs/2026-07-05-webhooks-design.md` (approved 2026-07-05).

**Architecture:** New `Webhook` + `WebhookDelivery` Prisma models (additive). New `webhook` API module (routes/service/repository, mirror saved-filter). Extend `activityService.record()` with fan-out (fetch workspaceId → find active webhooks → filter by events → enqueue BullMQ jobs). New `webhook` BullMQ worker reused inside `email-worker` container (no new docker service). Web `WebhooksTab` in settings.

**Tech Stack:** Prisma + Hono + Zod + BullMQ (existing), Node `crypto` HMAC-SHA256 (stdlib), `AbortController` timeout (stdlib). No new deps.

## Locked decisions (do not revisit)

- **D1 Worker:** reuse `email-worker` container, second BullMQ worker, separate `webhook` Redis queue.
- **D2 Emission:** `record()` enqueues directly; fetch `workspaceId` inside via `task.findUnique`.
- **D3 Delivery:** 5 attempts exp 2s, 10s timeout, 3 concurrency / 30-per-min. `X-FlowDesk-Signature: sha256=<hex>` + `X-FlowDesk-Event` + `X-FlowDesk-Delivery`. 5-state WebhookDelivery. No body snapshot.
- **D4 Model:** `Webhook` (workspaceId/url/secret/events[]/isActive/soft-delete), `WebhookDelivery` (Q3 shape). Admin+ mutate / Member+ read. Secret-once-reveal. `events` validated against `ActivityAction` enum. Mount `/api/workspaces/:wid/webhooks`.
- **D5 UI:** one `WebhooksTab` — list + create/edit Dialog + secret-reveal Dialog + delivery-log Dialog. Toggle Badge/Button for events + isActive. No new primitives.

## Global Constraints

- Schema hygiene (AGENTS.md §Future-Sprint): no `board` in names; additive migration; no Task field changes; filter-by-parameter.
- Module layout: `apps/api/src/modules/webhook/{webhook.routes,webhook.service,webhook.repository,webhook.test,index}.ts`; Zod in `packages/shared/src/webhook.ts`; web feature `apps/web/src/features/webhook/`.
- `Webhook` + `WebhookDelivery`: `id` (cuid), `createdAt`, `updatedAt`, `deletedAt?` (soft delete). `@@index` on FKs + common filters.
- Soft-delete: add both models to `SOFT_DELETE_MODELS` in `apps/api/src/shared/lib/prisma-extension.ts` AND `packages/db/src/prisma-extension.ts` (drift fixed in P1-2 — keep both in sync).
- Secret: 32-byte crypto random, hex. Returned ONLY on create response. Stripped from list/get/patch. Never logged.
- `events String[]`: Zod validates against `ActivityAction` enum at API boundary; stored as strings.
- `record()` fan-out is best-effort: existing try/catch, enqueue failure logs + returns null, doesn't throw.
- Webhook worker: `concurrency: 3`, `limiter: { max: 30, duration: 60_000 }`, `attempts: 5`, `backoff: { type: 'exponential', delay: 2000 }`, 10s `AbortController` timeout.
- HMAC-SHA256 via `crypto.createHmac('sha256', secret).update(body).digest('hex')`, prefix `sha256=`.
- No new deps. No `any`. No new docker container. No new tsup entry.
- Web test ships with feature (F8 pattern).
- One feature `in_progress` in `feature_list.json` at a time.

  ```ts
  import { Queue } from 'bullmq';
  import { env } from '../../shared/lib/env';
  export const WEBHOOK_QUEUE_NAME = 'webhook';
  export interface WebhookDeliveryJobData {
    webhookId: string;
    activityId: string;
    deliveryId: string;
  }
  export function createWebhookQueue() {
    return new Queue<WebhookDeliveryJobData>(WEBHOOK_QUEUE_NAME, {
      connection: { url: env.REDIS_URL },
      defaultJobOptions: {
        removeOnComplete: 1000,
        removeOnFail: 5000,
        attempts: 5,
        backoff: { type: 'exponential', delay: 2000 },
      },
    });
  }
  export const webhookQueue = createWebhookQueue();
  export async function enqueueWebhookDelivery(data: WebhookDeliveryJobData) {
    return webhookQueue.add('deliver', data, { jobId: data.deliveryId });
  }
  ```
- [ ] **5.2** `apps/api/src/workers/webhook/processors/deliver.ts`:
  ```ts
  const worker = new Worker<WebhookDeliveryJobData>(
    'webhook',
    async (job) => {
      const { webhookId, activityId, deliveryId } = job.data;
      const webhook = await prisma.webhook.findUnique({ where: { id: webhookId } });
      const activity = await prisma.taskActivity.findUnique({ where: { id: activityId } });
      if (!webhook || !activity) return; // deleted, skip
      const body = JSON.stringify({
        event: activity.action,
        deliveryId,
        timestamp: activity.createdAt.toISOString(),
        data: {
          activityId,
          taskId: activity.taskId,
          userId: activity.userId,
          field: activity.field,
          oldValue: activity.oldValue,
          newValue: activity.newValue,
          metadata: activity.metadata,
        },
      });
      const sig = `sha256=${crypto.createHmac('sha256', webhook.secret).update(body).digest('hex')}`;
      // update delivery: PROCESSING, attemptCount++
      await prisma.webhookDelivery.update({
        where: { id: deliveryId },
        data: { status: 'PROCESSING', attemptCount: { increment: 1 } },
      });
      const controller = new AbortController();
      const timeout = setTimeout(() => controller.abort(), 10_000);
      try {
        const res = await fetch(webhook.url, {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json',
            'X-FlowDesk-Signature': sig,
            'X-FlowDesk-Event': activity.action,
            'X-FlowDesk-Delivery': deliveryId,
          },
          body,
          signal: controller.signal,
        });
        clearTimeout(timeout);
        const responseBody = (await res.text()).slice(0, 1024);
        if (res.status >= 200 && res.status < 300) {
          await prisma.webhookDelivery.update({
            where: { id: deliveryId },
            data: {
              status: 'SUCCESS',
              responseCode: res.status,
              responseBody,
              deliveredAt: new Date(),
              error: null,
            },
          });
        } else {
          await prisma.webhookDelivery.update({
            where: { id: deliveryId },
            data: { responseCode: res.status, responseBody, error: `HTTP ${res.status}` },
          });
          throw new Error(`HTTP ${res.status}`); // → BullMQ retry
        }
      } catch (err) {
        clearTimeout(timeout);
        const msg = err instanceof Error ? err.message : String(err);
        await prisma.webhookDelivery.update({ where: { id: deliveryId }, data: { error: msg } });
        throw err; // → BullMQ retry (AbortError or network)
      }
    },
    { connection: { url: env.REDIS_URL }, concurrency: 3, limiter: { max: 30, duration: 60_000 } },
  );
  worker.on('failed', (job, err) => {
    /* on final attempt, mark FAILED/ERROR */
  });
  export const webhookWorker = worker;
  ```
  Handle final-attempt status: in `worker.on('failed', ...)`, check `job.attemptsMade >= job.opts.attempts` → update delivery status to `FAILED` (had responseCode) or `ERROR` (no responseCode, exception).

## Task 6 — Activity fan-out in `record()`

- [ ] **6.1** `apps/api/src/modules/activity/activity.service.ts` — extend `record()`:
  ```ts
  export async function record(input: RecordInput): Promise<TaskActivity | null> {
    try {
      const activity = await repo.create(prisma, input);
      // Webhook fan-out (best-effort, doesn't throw)
      try {
        const task = await prisma.task.findUnique({
          where: { id: input.taskId },
          select: { workspaceId: true },
        });
        if (task) {
          const webhooks = await prisma.webhook.findMany({
            where: { workspaceId: task.workspaceId, isActive: true, deletedAt: null },
          });
          const matching = webhooks.filter((w) => w.events.includes(input.action));
          for (const w of matching) {
            const delivery = await prisma.webhookDelivery.create({
              data: { webhookId: w.id, activityId: activity.id, status: 'PENDING' },
            });
            await enqueueWebhookDelivery({
              webhookId: w.id,
              activityId: activity.id,
              deliveryId: delivery.id,
            });
          }
        }
      } catch (fanOutErr) {
        logger.warn(
          { err: fanOutErr, taskId: input.taskId, action: input.action },
          'webhook fan-out failed',
        );
      }
      return activity;
    } catch {
      return null;
    }
  }
  ```
