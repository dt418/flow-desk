# RISKS — FlowDesk Risk Register

| #                | Risk                                                                                                         | Likelihood | Impact   | Mitigation                                                                                                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------------------------------ | ---------- | -------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| R-01             | **AI provider downtime** — OpenAI-compatible API is down or rate-limited, breaking AI features               | Medium     | High     | LLMProvider returns graceful fallback (rule-based suggestion); UI shows "AI unavailable" badge; feature is non-blocking                                                    |
| R-02             | **Socket.IO scale** — >1k concurrent users in workspace causes Redis pub/sub bottleneck                      | Low        | High     | Redis adapter handles horizontal scale; monitor with Prometheus; rate-limit per-user emits                                                                                 |
| R-03             | **JWT secret leak** — `JWT_SECRET` env var exposed via logs or git                                           | Low        | Critical | Secret rotated via init.sh; never logged; docker-compose reads from `.env` (gitignored); check git history on each commit                                                  |
| R-04             | **Soft-delete complexity** — Queries must always filter `deletedAt: null`; missing filter leaks deleted data | High       | Medium   | Prisma middleware `softDelete` extension; add test that asserts deleted records are not returned; lint rule for raw `prisma.findMany` without `where: { deletedAt: null }` |
| R-05             | **Password brute force** — Attacker tries many passwords on login endpoint                                   | High       | High     | Rate limit `/api/auth/login` to 5 req/min per IP+email; lockout after 10 failed attempts in 15min; log to security audit table                                             |
| R-06             | **File upload DoS** — User uploads huge file, exhausts disk                                                  | Medium     | Medium   | Max upload 25MB; per-user daily quota (1GB); use streaming multipart parser (busboy) not in-memory                                                                         |
| R-07             | **Circular dependency infinite loop** — A blocks B, B blocks A                                               | Medium     | Low      | Detect cycle at create time via DFS in service layer; return 400 with clear error                                                                                          |
| R-08             | **Real-time race condition** — User A and B both move same task; both writes succeed, last-write-wins        | Medium     | Medium   | Include `version` field on Task; service uses optimistic locking (UPDATE ... WHERE version = $1); emit conflict event so clients can re-sync                               |
| R-09             | **Mention notification spam** — Comment with @everyone@everyone@... creates N notifications                  | Low        | Medium   | Parse @mentions with dedup set; cap notifications per comment to 10; rate-limit mention sends per user per hour                                                            |
| R-09 (mitigated) | Same as above                                                                                                | —          | —        | `comment.service.ts`: dedup via `Set` + cap `MAX_MENTIONS=10` (commit session 020)                                                                                         |
| R-10             | **Drag-drop UX on mobile** — dnd-kit desktop-first, mobile gestures awkward                                  | Medium     | Medium   | Use dnd-kit's touch sensor config; test on iOS Safari; fallback to long-press menu on small screens                                                                        |
| R-10 (mitigated) | Same as above                                                                                                | —          | —        | `kanban.tsx`: `TouchSensor` added with `{ delay: 200, tolerance: 5 }` (commit session 020)                                                                                 |
| R-11             | **Cookie CORS misconfig** — Frontend on :5173, API on :3000, cookies not sent                                | High       | High     | Backend CORS: `origin: ['http://localhost:5173', 'http://localhost:3000']`, `credentials: true`; frontend `fetch`: `credentials: 'include'`; document in ADR-003           |
| R-12             | **Migration breaks production** — New migration drops column, data loss                                      | Low        | Critical | All migrations are additive; deprecate columns first (mark @deprecated, ignore in code), drop only after 2+ releases; always provide `down` migration script               |
| R-13             | **Tailwind v4 alpha instability** — v4 is recent; breaking changes likely                                    | Medium     | Medium   | Pin exact version; subscribe to Tailwind v4 release notes; lockfile committed; shadcn/ui compatibility check before upgrade                                                |
| R-14             | **PostgreSQL connection exhaustion** — Too many API instances open too many connections                      | Medium     | High     | Prisma connection pool size = 10 per instance; PgBouncer in front of Postgres in docker-compose (Phase 2); monitor with `pg_stat_activity`                                 |
| R-15             | **AI hallucination** — LLM suggests invalid schedule or wrong assignee                                       | High       | Low      | Always show AI suggestions as suggestions, not actions; user must confirm; never auto-apply destructive AI decisions                                                       |
| R-16             | **Soft delete cascade** — Deleting workspace should soft-delete tasks but not orphan comments                | Medium     | Medium   | Soft-delete at workspace level only; tasks/comments retain deletedAt set on cascade; queries always filter; data recoverable within 30 days via admin tool                 |
| R-17             | **OAuth state CSRF** — Forged OAuth callback creates session for attacker                                    | Low        | High     | OAuth `state` param with random nonce stored in session; verify on callback                                                                                                |
| R-18             | **Time zone bugs** — Due dates stored in UTC, displayed wrong for user                                       | High       | Medium   | Store all timestamps UTC; UI formats with `Intl.DateTimeFormat` using user locale + timezone; never use `new Date()` without timezone                                      |
| R-18 (mitigated) | Same as above                                                                                                | —          | —        | `format-date.ts`: `formatDate` and `relativeDays` accept optional `timeZone` param; defaults UTC-safe `Intl` locale fallback retained; callers can pass user timezone      |

| R-19 | **Prisma N+1** — List tasks with assignees triggers N+1 queries | High | Medium | Use `include: { assignee: true }`; check query plan in test; add `@@index` on all FKs |
| R-20 | **Self-hosted upgrade path** — User upgrades v1.0 → v1.1 with breaking schema change | Medium | High | Versioned migrations; `pnpm migrate deploy` runs additive migrations on startup; never auto-drop data; document manual migration steps in CHANGELOG |
| R-21 | **Settings UI role bypass** — Member tampers with role-change select to demote Owner | Low | High | UI hides role-change select for non-Owners; backend `requireWorkspaceRole(['OWNER'])` on PATCH `/members/:userId`; last-Owner demotion blocked at service layer |
| R-22 | **Settings UI stale role** — User's cached role in `['workspaces']` query out of sync with server | Medium | Low | `useWorkspaceRole` reads from `['workspaces']` cache; mutations that change role invalidate `['workspaces']` via `onSuccess`; Socket.IO broadcasts trigger `qc.invalidateQueries(['workspaces'])` |
| R-23 | **Confirm-by-name typo** — User mistypes workspace name and deletes the wrong workspace | Low | High | Exact-match required (trimmed) before Delete enables; soft-delete (deletedAt) so 30-day recovery via admin tool |
| R-24 (mitigated) | **AI suggest latency** — Slow LLM proxy made UI hang on Suggest | High | Medium | 5s soft timeout on suggest LLM call (no retry); rule-based fallback with `fallbackReason: timeout\|error`; FE cancel + elapsed "AI thinking… Ns"; abort on modal close/unmount; `llmLatencySeconds` metrics |
| R-25 (resolved) | **Socket.IO zero emissions** — rooms joined but no `io.to().emit()` anywhere | High | High | F1: `socket-events` singleton + emit helpers in every route mutation; verified via live curl `/socket.io/?EIO=4&transport=polling` |
| R-26 (resolved) | **No rate limiting** — auth/AI/write paths unprotected | High | High | F1: `rate-limit` middleware (Redis INCR+EXPIRE), per-scope limits (auth: 3/h-5/min-30/min, AI 5/min/user, write 60/min/user), `X-RateLimit-*` + `Retry-After` headers |
| R-27 (resolved) | **Attachment IDOR** — `GET /api/attachments/:id/download` no membership check | High | Critical | F1: `assertMembership` on all 3 attachment endpoints |
| R-28 (resolved) | **Membership missing on AI + comment + attachment POST** | High | High | F1: `assertMembership` helper extracted to `shared/lib/access.ts`, applied to all mutation routes |
| R-29 (resolved) | **Soft-delete gaps** — `PATCH /workspaces/:id`, dependency endpoints, AI task lookup, comment-task lookup, attachment upload allow operations on deleted entities | Medium | Medium | F3: 12 gaps audited+fixed across 6 modules; Prisma `softDeleteExtension` auto-filters findFirst/findMany/count/aggregate/groupBy on 6 models; 15 integration tests pass |
| R-30 (resolved) | **Missing pagination** — workspaces list, members list, attachments list, board columns (hardcoded `take:50`) | Medium | Low | F4: `CursorPaginationQuery` + `CursorPaginationEnvelope` in packages/shared/src/pagination.ts; 7 list endpoints standardized; 12 pagination tests pass |
| R-31 (resolved) | **Zero service/repository layer** — every backend module is a single fat routes.ts with inline Prisma; impossible to unit-test business logic | Medium | Medium | F2 (label/task-label/workspace) + F5 (task/comment/notification/attachment/ai): all modules now follow {feature}.{routes,service,repository,schema}.ts pattern |
| R-32 (resolved) | **Zero tests** — `**/*.test.ts` empty across the repo | High | Medium | F2 (42 tests) + F3 (15 soft-delete) + F4 (12 pagination) + F5 (73 across 5 modules): total 142 integration tests passing |
| R-33 (resolved) | **Split-brain selects** — `components/ui/select.tsx` (Radix) unused; `list.tsx` + GeneralTab + MembersTab + NewTaskModal all use native `<select>` | Low | Low | F2: shadcn dropdown-menu/popover/radio-group wired for label color picker, member invite role, presence overflow |
| R-34 (resolved) | **DragOverlay UX** — static "Moving…" instead of card clone; no `SortableContext` → cards teleport on drop | Low | Low | F2 added snapshot+rollback; F6 added renderOverlay prop on Kanban — DragOverlay now renders real TaskCard clone during DnD |
| R-35 (resolved) | **Pre-existing src/index.ts typecheck error** — `ServerType` (Http2\|Http1 union) not assignable to `Server<Http1>` from `@hono/node-server`; confirmed pre-existing via `git stash` + typecheck | Low | Low | Backlog: cast or narrow `ServerType` at `createSocketServer` call site; deferred — does not block runtime (smoke verify passes) |
| R-36 (resolved) | **Click bubbling on nested interactive elements** — kebab menu + label select on task card may not fully block event propagation to card's onClick, causing unintended navigation or modal opens | High | High | Sprint 1: INTERACTIVE_SELECTOR + NoCardClick wrapper; kebab + label select wrapped in NoCardClick to stop propagation before it reaches card handler |
| R-37 (resolved) | **PointerSensor 80ms click lag** — `{distance:6, delay:80}` forces every click to hold 80ms before firing, making board feel sluggish | High | Medium | Sprint 1: PointerSensor {distance:8} with no delay; TouchSensor {delay:150, tolerance:8} for mobile |
| R-38 (resolved) | **Nested role=button axe violation** — useDraggable spreads {...attributes} onto wrapper div while inner article also has role='button', creating nested interactive axe violation | High | Medium | Sprint 1: attributes+listeners spread on inner div only; article uses aria-roledescription='draggable' + aria-label instead of role='button' |
| R-39 | **Optimistic reorder + Socket.IO self-broadcast race** — after drop, Socket.IO broadcasts task:moved back to the sender, invalidating the board query before the mutation response arrives, causing a flash/revert | High | High | Sprint 1.5: move-progress flag skips task:moved invalidation during in-flight mutation; onSettled handles final invalidation |
| R-40 | **Same-position move fires unnecessary API call** — dropping a card back to its original position triggers a full move mutation with position reorder | Low | Low | Sprint 1.5: early-return in handleMove when fromColumnId === toColumnId && fromIndex === toIndex |
| R-41 | **DragOverlay invisible snap** — isDragging instantly sets card invisible, creating a jarring visual handoff to the DragOverlay | Medium | Low | Sprint 1.5: isDragging uses opacity-30 + transition-opacity duration-150 for smooth fade |
| R-42 | **Schema painted into corner** — phase 1–2 query/naming bakes in `workspace == board`, blocking Epic/Sprint/Board models later and forcing a rewrite of every board query | Medium | High | AGENTS.md "Future-Sprint Schema Hygiene" checklist: no `board` in names, structural fields limited to `columnId`+`parentTaskId`, filter-by-parameter signatures, Epic via `parentTaskId` reuse, Sprint+estimation deferred together |
| R-43 (resolved) | **softDeleteExtension drift** — packages/db/src/prisma-extension.ts missing ChatChannel/ChatMessage/SavedFilter from SOFT_DELETE_MODELS; module-level prisma singleton (used by all routes) did not filter soft-deleted records for these models while test prisma (apps/api copy) did | High | High | P1-2 Task 6: synced packages/db copy to match apps/api copy; pre-existing bug from F7 chat work; all 3 models now in SOFT_DELETE_MODELS |
| R-44 | **Partial unique index not expressible in Prisma schema** — SavedFilter name uniqueness (per user per workspace, ignoring soft-deleted) requires `CREATE UNIQUE INDEX ... WHERE deletedAt IS NULL`; Prisma `@@unique` cannot express partial indexes; raw SQL in migration only | Medium | Low | Manual `CREATE UNIQUE INDEX` in migration SQL; schema.prisma cannot validate; future schema resets must re-apply via migration deploy |
| R-45 (mitigated) | **Chat public-channel IDOR** — `findAndValidateChannel` only checked membership for private channels | High | High | Audit 029: always assert workspace membership; typing requires `conversation:join`; IDOR integration tests |
| R-46 (mitigated) | **Google OAuth skipped 2FA; Slack slash unsigned; integration callback required query workspaceId** | High | High | Audit 030: httpOnly `two_factor_challenge` cookie + `?twoFactor=1`; Slack HMAC raw body; cookie-only workspaceId |
| R-47 (mitigated) | **Outbound webhook/automation SSRF** — `fetch` to admin-controlled URLs without private/metadata deny or DNS pin | High | High | Audit 032 + review: `url-safety` + undici DNS pin; always block loopback/169.254/IMDS IPv6; optional `ALLOW_PRIVATE_WEBHOOK_URLS` |
